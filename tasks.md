Accessor Dashboard Inputs
Claim Overview –
(i) Claim ID / Reference Number
(ii) Date of Submission
(iii) Status (Pending Review / Under Assessment / Approved / Flagged / Rejected)
(iv) Assigned Assessor Name & ID
User & Vehicle Info (Auto-fetched from claim)
(i) Policy Holder Name
(ii) Driving Licence Number
(iii) Vehicle Make & Model
(iv) Vehicle Colour
(v) Policy Number
Incident Data Review –
(i) Date, Time & Location of Incident
(ii) Situation Type (Accident, Theft, Fire, etc.)
(iii) Other Party Details
(iv) Licence Plate
(v) Reported Injuries (if any)
Visual Evidence & Analysis –
(i) Uploaded Pictures
(ii) 3D Model View of Vehicle with Damage Indicators (generated via AI)
(iii) Image Metadata Verification (time, location, authenticity)
(iv) Additional File Attachments (police report, witness statements)
AI Fraud Intelligence Insights –
(i) Claim Authenticity Score (0–100%)
(ii) Damage Severity Score
(iii) Anomaly Detection Summary (e.g., inconsistent timestamps, reused photos)
(iv) Cross-Claim Similarity Check (duplicate or suspicious claims)
Manual Assessment Section –
(i) Assessor’s Comments
(ii) Estimated Repair Cost (auto-filled + editable)
(iii) Recommended Claim Amount
(iv) Fraud or Error Flags (checkboxes: image tampering, inconsistent story, etc.)
Final Decision –
(i) Approve / Reject / Request More Info
(ii) Attach Assessment Report (PDF)
(iii) Timestamp of Final Decision
(iv) Forward to Next Authority (if required)
